# 均衡結合モデル — 数値スケール・評価式

出典: Vlad Khononov『ソフトウェア設計の結合バランス』（Balancing Coupling in Software Design）。
以下は同書の均衡結合モデルを本スキル用に数値化した定義集。スコアはすべて 1〜10。
数値の割り当て（強度スコア等）は本スキル独自の運用値であり、書籍が示すのは順序関係である。

---

## 3つの結合次元

| 次元 | 意味 | 本スキルでの決定方法 |
|---|---|---|
| **統合強度 (Strength)** | 結合した2コンポーネントが共有する「知識」の質。強いほど片方の変更が他方に波及する | ワーカーが実ソースを読んで確定（`strength-criteria.md`） |
| **距離 (Distance)** | コードベース内での配置の遠さ。遠いほど共同変更の調整コストが大きい | リポジトリ構造から機械的に算出 |
| **変動性 (Volatility)** | 将来変更される予想頻度。高いほど結合が問題化しやすい | git 実績 + ドメイン分析の2系統 |

**上流/下流**: A が B に依存するとき B が上流、A が下流。知識は上流→下流へ流れる（依存の向きと逆）。

## フラクタル性 — 粒度非依存の根拠（第12章）

均衡結合モデルは自己相似的で、メソッド間・クラス間・パッケージ間・モジュール間・サービス間のどの粒度にも同じ式が適用できる。粒度が変わるのは距離の絶対値だけで、強度・変動性の判定方法と評価式は不変。本スキルが評価粒度を自動推定できるのはこの性質による。

## 統合強度スケール（第7章、詳細は strength-criteria.md）

| レベル | スコア | 共有する知識 |
|---|---|---|
| 侵入結合 (Intrusive) | 10 | 上流の実装詳細（最悪） |
| 対称機能結合 | 9 | 同じ機能を両者が独立実装（DRY違反） |
| 機能結合 (Functional) | 8 | ビジネスルール・振る舞い・実行順序・トランザクション |
| モデル結合 (Model) | 3 | ビジネスドメインのデータモデル（構造のみ） |
| コントラクト結合 (Contract) | 1 | 統合専用モデルのみ（最良） |

レベル内はコナーセンスの度合いで微調整する（strength-criteria.md 参照）。

## 距離スケール（第8章）

距離の本質は「共同での変更に必要なコミュニケーション・調整コスト」。一般的な評価法は**最も近い共通祖先**の深さ: 共通祖先が深い（近い階層にある）ほど距離が近い。

書籍の例示スケール: 同一クラス内=1、同一パッケージ=2、パッケージ間=3〜7、別ライブラリ=8、別サービス=9、別ベンダー=10。

**運用上の注意**: 評価粒度を固定すると例示スケールでは全エッジが同じ距離になり、距離の次元が判別力を失う。そこで本スキルは共通祖先法をリポジトリ内の階層に適用し、リポジトリ専用のマッピング表（3〜9 に分布）を作って使う。絶対値は書籍の例と比較できない（リポジトリ内の相対比較専用）。

追加要因（第8.2節）: 所有チームが異なる（ソシオテクニカルな距離、コンウェイの法則）場合は実質的距離が遠い。根拠があれば +1。非同期通信はライフサイクル結合を弱めるが、統合強度が高ければ変更コストは下がらないので一律加点しない。

## 変動性スケール（第9章）

| 変動性 | サブドメイン種別 / 状態 |
|---|---|
| 1 | レガシー（進化を放棄・凍結） |
| 3 | 支援・汎用サブドメイン（安定） |
| 10 | コアサブドメイン（競争優位の源泉。継続的に進化し本質的に高変動） |

git 実績値（直近12ヶ月コミット数のパーセンタイルを 1-10 化）と**併記**する。

### 偽陽性・偽陰性の補正（9.3.2）
- **偽陽性**: fix 系コミットばかりで仕様は安定 → git 上は高頻度でも変動性を下げる。
- **偽陰性**: コアサブドメインなのに低コミット → 「変更困難ゆえ凍結」の疑い。安定と即断せず**フラグを立てる**（リファクタ優先度はむしろ上げる）。

### 実効変動性の伝播（9.4〜9.5）

上流の高変動性は統合強度が高いほど下流へ伝播する:

```
実効変動性 = max(基礎変動性, 上流変動性 × 減衰係数(統合強度))
```

| 統合強度 | 減衰係数 |
|---|---|
| 侵入結合(10) | ≈ 1.0（ほぼそのまま被る） |
| 機能結合(8-9) | ≈ 0.8 |
| モデル結合(3) | ≈ 0.4 |
| コントラクト結合(1) | ≈ 0.1（ほぼ伝播しない） |

エッジの変動性 V には**上流コンポーネントの実効変動性**を使う（下流が被る変更頻度だから）。

## 評価式（第10章）

強度 S / 距離 D / 変動性 V はすべて 1-10。**2値式の高低判定**: S≧6 / D≧6 / V≧6 を「高」とする。

### 均衡度 (Balance) ★リファクタ優先度の主指標
```
均衡度 = max(|S - D|, 10 - V) + 1     (1-10、高いほど良い)
```
ブール形: `均衡 = (強度 XOR 距離) OR NOT 変動性`。モジュール性が高いか変動性が低ければ均衡。**低いエッジほど優先度が高い。**

### モジュール性 (Modularity)
```
モジュール性 = |S - D| + 1            (1-10、高いほど良い)
```
高強度×近距離、または低強度×遠距離ならモジュール的。

### 複雑性 (Complexity)（2値）
- **大域的複雑性** = 高強度 AND 遠距離（S≧6 かつ D≧6）
- **局所的複雑性** = 低強度 AND 近距離（S<6 かつ D<6）= 無関係なものの同居

### 安定性 (Stability)（2値）
```
安定性 = NOT (高変動 AND 高強度)      V≧6 かつ S≧6 なら「不安定」
```

### 変更コスト (Cost of Change)（2値）
```
変更コスト = 高変動 AND 遠距離        V≧6 かつ D≧6 なら「高」
```

### メンテナンス労力 (Maintenance Effort)
```
労力 = S × D × V                      (生値。同点の順位付けに使う)
```
