英語の技術記事を translator / reviewer.english / reviewer.japanese の3エージェントで協調して日本語に変換するスキル。

## 入力
$ARGUMENTS

原文の指定方法:
- **URL**: `https://...` の形式
- **ローカルファイル**: 絶対パスまたは相対パス
- **インライン**: 上記以外はメッセージ内の貼り付けテキストと判断

---

## 作業ディレクトリ

`docs/translations/YYYY-MM-DD-<slug>/` を作業ディレクトリとする。以下のファイルを配置:

- `source.md` — 原文キャッシュ
- `structure.md` — reviewer.english による構造分析
- `translation.md` — 最新の訳文
- `review-en-N.md` — N 回目の reviewer.english レビュー結果
- `review-ja-N.md` — N 回目の reviewer.japanese レビュー結果

`YYYY-MM-DD` は実行日、`<slug>` はタイトルをケバブケース化。

---

## Phase 1: 原文のキャッシュ

原文をローカルの `source.md` に保存する。作業が進むにつれて元の原文がコンテキストから外れる可能性があるため、**エージェント呼び出し時には毎回ローカルキャッシュのパスを渡す**前提。

### URL の場合

1. まず WebFetch を試す
2. 失敗した場合、ドメインに応じた専用スキルを利用:
   - `medium.com` → fetch-medium スキル
   - その他 → Claude in Chrome MCP で取得
3. 取得したテキストを `source.md` に書き出す

### ローカルファイルの場合

Read で読み込み、作業ディレクトリの `source.md` にコピー（パス正規化のため）。

### インラインテキストの場合

直接 `source.md` に書き出す。

### キャッシュ後の確認

`source.md` の行数・最初の数行・最後の数行を確認し、取り込みが正常であることを検証する。欠損があれば再取得する。

## Phase 2: 構造分析

reviewer.english エージェントを起動し、原文の構造を分析する。

### Agent 呼び出し

- **subagent_type**: `reviewer.english`
- **description**: `原文構造分析`
- **prompt**:

```
フェーズ A (原文構造分析) を実施してください。

原文ファイル: <作業ディレクトリ>/source.md

以下を抽出し、`<作業ディレクトリ>/structure.md` に書き出してください:
- アウトライン（見出し階層）
- 論旨（3-5 文）
- キーフレーズ・専門用語
- トーン・文体（皮肉・ユーモアの有無）
- ニュアンス注意箇所

出力フォーマットはエージェント定義のフェーズ A セクションに従う。
```

---

## Phase 3: 翻訳

translator エージェントを起動し、構造分析を踏まえて全文翻訳する。

### Agent 呼び出し

- **subagent_type**: `translator`
- **description**: `初回翻訳`
- **prompt**:

```
以下の英文を日本語に翻訳してください。

原文ファイル: <作業ディレクトリ>/source.md
構造分析: <作業ディレクトリ>/structure.md
出力先: <作業ディレクトリ>/translation.md

必ず全文を翻訳してください。要約・省略は禁止です。
見出し・箇条書き・コードブロック等の構造は原文と揃えてください。
```

---

## Phase 4: レビュー（並列）

reviewer.english と reviewer.japanese を**同時に**起動する（1メッセージ内で2つの Agent tool use）。

N はループ回数（初回は 1）。

### SubAgent A: 英語話者視点レビュー

- **subagent_type**: `reviewer.english`
- **description**: `英語話者視点レビュー`
- **prompt**:

```
フェーズ B (訳文レビュー) を実施してください。

原文: <作業ディレクトリ>/source.md
和訳: <作業ディレクトリ>/translation.md
構造分析: <作業ディレクトリ>/structure.md
出力先: <作業ディレクトリ>/review-en-<N>.md

意味の逆転・ニュアンス崩れ・誤訳・誤解を招く表現を重大/中程度/軽微に分類して指摘してください。
```

### SubAgent B: 日本語話者視点レビュー

- **subagent_type**: `reviewer.japanese`
- **description**: `日本語話者視点レビュー`
- **prompt**:

```
和訳を日本語単体でレビューしてください。原文との照合は不要です。

和訳: <作業ディレクトリ>/translation.md
構造分析: <作業ディレクトリ>/structure.md
出力先: <作業ディレクトリ>/review-ja-<N>.md

不自然な翻訳調・矛盾・用語不統一などを重大/中程度/軽微に分類して指摘してください。
ユーモア・皮肉については直訳を維持し、自然化を強要しないでください。
```

---

## Phase 5: 指摘反映

両レビュー結果を読み、translator エージェントで反映する。

### Agent 呼び出し

- **subagent_type**: `translator`
- **description**: `レビュー指摘反映`
- **prompt**:

```
レビュー指摘を訳文に反映してください。

和訳（編集対象）: <作業ディレクトリ>/translation.md
英語話者視点レビュー: <作業ディレクトリ>/review-en-<N>.md
日本語話者視点レビュー: <作業ディレクトリ>/review-ja-<N>.md
原文: <作業ディレクトリ>/source.md
構造分析: <作業ディレクトリ>/structure.md

指摘を一件ずつ反映し、修正後に原文構造から乖離していないか自己チェックしてください:
- 章立てが変わっていないか
- 意味の逆転が生じていないか
- 省略が発生していないか

修正後の訳文で translation.md を上書きしてください。
```

---

## Phase 6: 改善ループ判定

Phase 4 〜 5 を繰り返す。以下のいずれかで終了:

1. **指摘なしまたは軽微のみ**: 両方のレビュー結果で「重大」「中程度」がなく、軽微の指摘だけになった時点で終了
2. **ループ回数が 3 回に達した**: 品質が収束しない場合の無限ループ防止

ループごとに N をインクリメントし、`review-en-N.md` / `review-ja-N.md` として新規に保存する（上書きしない）。

### 終了判定の具体的基準

各レビュー結果ファイルの冒頭セクションを確認し:

- `## 重大` セクションに項目がある → 継続
- `## 中程度` セクションに項目がある → 継続
- 上記両方が空または存在しない → 終了候補
- 両レビューとも終了候補 → 終了

---

## Phase 7: 完了報告

ユーザーへ以下を報告:

- 作業ディレクトリのパス
- 最終訳文のパス（`translation.md`）
- ループ回数と各ラウンドのサマリ（重大/中程度/軽微の件数推移）

---

## ルール

- **全文翻訳**: 要約・省略・抜粋は禁止。
- **並列レビュー**: Phase 4 は必ず1メッセージで2エージェントを同時起動。
- **ローカルキャッシュ優先**: エージェント呼び出し時は毎回ローカルファイルパスを渡す。インラインで原文を貼り付けない。
- **ファイル名のバージョニング**: `translation.md` は上書きだが、レビュー結果は世代ごとに別ファイル。差分を追えるようにする。
