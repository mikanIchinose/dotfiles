このスキルは、インターフェースやクラスの変更前に影響範囲を調査し、修正漏れを防止します。

## トリガー

- 「影響範囲を調査して」
- 「〇〇の実装クラスを探して」
- 「〇〇を使っている箇所を調べて」
- `/impact-analysis InterfaceName`

## 手順

### 1. 変更対象の特定

ユーザーに以下を確認:
- **変更対象**: インターフェース名 / クラス名
- **変更種別**: メソッド追加 / コンストラクタパラメータ追加 / その他

### 2. 影響範囲の検索

変更種別に応じて Grep を実行:

#### インターフェースにメソッドを追加する場合

```
Grep ": {InterfaceName}" --type kt
Grep "implements {InterfaceName}" --type java
```

検索対象:
- 本番コード (`src/main`)
- テストコード (`src/test`, `src/androidTest`)
- Fakeクラス、Mockクラス、Stubクラス

#### クラスのコンストラクタにパラメータを追加する場合

```
Grep "{ClassName}(" --type kt
```

検索対象:
- インスタンス生成箇所
- DI設定（Module、Provider）
- テストコード内のインスタンス生成

### 3. 結果の整理

検索結果から以下を抽出:
- ファイルパス
- クラス名
- 用途（本番/テスト/Fake等）

### 4. TODOリストへの追加

TodoWrite ツールを使用して、以下の形式でTODOを追加:

```
- [ ] {InterfaceName} の影響範囲調査 ← 完了としてマーク
- [ ] {InterfaceName} に変更を加える
- [ ] {ファイル1} を修正
- [ ] {ファイル2} を修正
- [ ] ... (全ファイル分)
- [ ] ビルド確認
- [ ] テスト実行
```

## 出力フォーマット

```markdown
## 影響範囲調査結果: {変更対象名}

### 変更種別
{メソッド追加 / コンストラクタパラメータ追加 / その他}

### 検出されたファイル

#### 本番コード
| ファイル | クラス名 | 説明 |
|----------|----------|------|
| path/to/File.kt | ClassName | 実装クラス |

#### テストコード
| ファイル | クラス名 | 説明 |
|----------|----------|------|
| path/to/FakeClass.kt | FakeClassName | テスト用Fake |

### TODOリスト
上記ファイルをTODOリストに追加しました。順次修正してください。
```

## 注意事項

- 検索結果が0件の場合でも、検索コマンドと結果を報告する
- 検索パターンに漏れがないか、ユーザーに確認を促す
- 大量のファイルがヒットした場合は、グループ化して報告する
