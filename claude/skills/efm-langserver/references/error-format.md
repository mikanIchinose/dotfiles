# Error Format リファレンス

efm-langserver の `lint-formats` は Vim の `errorformat` と同じ仕様を使う。
C 言語の `scanf` に似ているが、Vim 独自の拡張がある。

## 基本的な変換文字

| 文字 | 意味 | 例 |
|------|------|-----|
| `%f` | ファイル名 | `main.sh`, `-`(stdin) |
| `%l` | 行番号 | `42` |
| `%c` | 列番号 | `6` |
| `%t` | エラータイプ（1文字） | `e`(error), `w`(warning) |
| `%n` | エラー番号 | `2086` |
| `%m` | メッセージ（残り全部） | `Double quote to prevent...` |
| `%%` | リテラルの `%` | |

## 読み飛ばし

| パターン | 意味 | 用途 |
|----------|------|------|
| `%*[^X]` | 文字 `X` 以外を読み飛ばす | severity の残り部分をスキップ等 |
| `%*[chars]` | `chars` に含まれる文字を読み飛ばす | |
| `%.%#` | 任意の文字列（`.*` 相当） | 不要な部分をスキップ |

## 複数行対応

| 文字 | 意味 |
|------|------|
| `%A` | 複数行メッセージの開始行 |
| `%C` | 複数行メッセージの継続行 |
| `%Z` | 複数行メッセージの終了行 |
| `%E` | 複数行エラーの開始行 |
| `%W` | 複数行警告の開始行 |

## 特殊パターン

| 文字 | 意味 |
|------|------|
| `%>` | 次のパターンから再マッチを開始 |
| `%-G` | マッチした行を無視（ヘッダ・フッタのスキップ用） |
| `%+G` | マッチした行をメッセージ全体として使用 |

## よくあるパターン集

### ファイル名:行番号: メッセージ
```
出力: main.sh:10: some error
パターン: %f:%l: %m
```

### ファイル名:行番号:列番号: メッセージ
```
出力: main.sh:10:5: some error
パターン: %f:%l:%c: %m
```

### gcc 互換フォーマット
```
出力: main.sh:10:5: warning: some message
パターン: %f:%l:%c: %t%*[^:]: %m
```

`%t` は `w` (warning の頭文字) をキャプチャし、`%*[^:]` が `arning` を読み飛ばす。

### severity が大文字の場合
```
出力: main.sh:10:5: Warning: some message
パターン: %f:%l:%c: %t%*[^:]: %m
```

`%t` は大文字小文字どちらも 1 文字キャプチャする。efm-langserver が適切に解釈する。

### 行番号のみ（列番号なし）
```
出力: main.sh:10 some message
パターン: %f:%l %m
```

### エラー番号付き
```
出力: main.sh:10:5: error E001: some message
パターン: %f:%l:%c: %t%*[^ ] %m
```

### パス内にスペースがない前提（stdin の `-` を含む）
```
出力: -:1:6: warning: Double quote to prevent globbing [SC2086]
パターン: %f:%l:%c: %t%*[^:]: %m
```

## パターン作成の手順

1. ツールを実際に実行して出力を得る
2. 出力の各部分を変換文字に対応させる
3. 不要な部分は `%*[^X]` や `%.%#` で読み飛ばす
4. 複数のフォーマットがある場合は `lint-formats` に複数パターンを列挙（上から順にマッチを試行）

## 複数パターンの例

ツールが複数の出力形式を持つ場合:

```yaml
lint-formats:
  - '%f:%l:%c: %t%*[^:]: %m'   # 列番号あり
  - '%f:%l: %t%*[^:]: %m'      # 列番号なし
```

上から順に試行され、最初にマッチしたパターンが使われる。

## デバッグのコツ

- まずツールの出力を正確に把握する（`echo ... | tool` で確認）
- gcc 互換出力オプション（`-f gcc`, `--format=gcc` 等）があれば使う
- `%m` は行末まで全てキャプチャするので、パターンの最後に置く
- パターンがマッチしない場合、空白やコロンの位置を再確認する
