# ADR 作成スキル

## 手順

### 1. 既存ADRの確認

```bash
ls docs/adr/*.md | sort
```

最新の番号を確認し、次の番号を決定。

### 2. ADRテンプレート

```markdown
# ADR-NNNN: [タイトル]

## ステータス

[提案 | 採用 | 非推奨 | 却下]

## コンテキスト

[この決定が必要になった背景・状況]

### 前提条件

- [関連する前提や制約]

## 決定

### 1. [決定事項1]

[詳細な説明、コード例など]

### 2. [決定事項2]

[詳細な説明]

## 理由

1. [理由1]
2. [理由2]

## 結果

- [この決定による影響・結果]
```

### 3. ファイル作成

```bash
# 例: 5番目のADR
touch docs/adr/0005-[slug].md
```

ファイル名規則: `NNNN-kebab-case-title.md`

### 4. 関連ADRの参照

必要に応じて既存ADRを参照し、一貫性を保つ。

```bash
# 関連しそうなADRを確認
grep -l "キーワード" docs/adr/*.md
```

## ステータスの意味

| ステータス | 説明 |
|-----------|------|
| 提案 | 議論中、まだ確定していない |
| 採用 | 決定済み、有効 |
| 非推奨 | 以前は採用されていたが、現在は推奨しない |
| 却下 | 検討したが採用しなかった |

## 注意事項

- 既存ADRとの整合性を確認
- コード例は実際に動作するものを記載
- 「なぜその決定をしたか」を重視
